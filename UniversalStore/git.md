🔧 Первоначальная настройка Git (один раз)
Эти настройки нужно сделать на любом компьютере, где вы работаете с Git, до того, как вы сделаете первый коммит.

Откройте терминал (в Windows можно использовать Git Bash или PowerShell).

Введите по очереди две команды, заменив "Ваше Имя" и "ваша@почта.com" на свои данные:

bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваша@почта.com"
Флаг --global означает, что эти настройки будут использоваться для всех ваших проектов на этом компьютере.

Если Git не настроить, он не даст сделать коммит и попросит это исправить.

🚀 Работа в PyCharm
После этого никаких дополнительных регистраций не нужно:

Интеграция с Git: PyCharm обычно сам находит установленный Git. Вы можете проверить это в настройках: File → Settings → Version Control → Git. В поле Path to Git executable должен быть указан корректный путь, а кнопка Test должна показать версию. Если пути нет, PyCharm, скорее всего, сам предложит его настроить при первом обращении к Git-функциям.

Как начать: Самый простой способ начать версионный контроль в PyCharm — через меню VCS → Enable Version Control Integration... и выбрать в выпадающем списке Git. PyCharm создаст репозиторий в корне вашего проекта.

Как только репозиторий инициализирован, вы можете выполнять все Git-операции:

Через интерфейс PyCharm: Использовать меню Git, контекстное меню файлов или панель Commit (которая появляется после Ctrl+K или Cmd+K).

Через терминал: Просто открыть встроенный терминал PyCharm (кнопка Terminal внизу) и вводить знакомые команды: git add ., git commit -m "...". Это абсолютно равноправный способ.

💎 Что в итоге
Ваш план на экзамене будет простым:

Убедиться, что Git (именно программа, а не плагин в PyCharm) настроен глобально.

Открыть проект в PyCharm.

Начать работать с Git любым удобным способом: хоть через интерфейс IDE, хоть через терминал — оба подхода корректны.


Как правило, он выглядит так:

Получаете ссылку на репозиторий: В начале экзамена вы получаете персональную ссылку (обычно на GitHub Classroom, GitLab или аналогичную платформу).

Копируете проект к себе: В терминале Git (консоль Git Bash) пишете команду git clone [ссылка_на_ваш_репозиторий]. После этого у вас на компьютере появится локальная копия проекта.

Переносите код в эту папку: Дальше вы просто копируете все файлы вашего проекта (тот самый main.py, database.sql, README.md и папку images) в скачанную на предыдущем шаге папку.

Фиксируете и отправляете работу: После этого выполняете стандартную цепочку Git-команд, чтобы ваши изменения попали в «облачную» систему экзаменаторов:

bash
git add .                     # 1. Добавляете все файлы
git commit -m "Готовое решение" # 2. Фиксируете изменения
git push origin main          # 3. Отправляете их на проверку
Вот и весь алгоритм. Все эти команды мы с вами уже разбирали, они вам знакомы. Самое важное — это действовать внимательно и по шагам.

Главное — не стесняйтесь ничего уточнять. Если возникнут новые вопросы, смело задавайте. Удачи на экзамене!
